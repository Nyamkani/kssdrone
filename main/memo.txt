(gpio_set_drive_capability((gpio_num_t)3, GPIO_DRIVE_CAP_3); )
3.3V 신호의 파형 끝(Edge)이 더 날카로워져서,
 47옴 저항 뒤에 있는 ESC가 신호를 인식하는 성공률이 크게 올라갈 것입니다.

1ms 루프 (1kHz): 1ms 단위로 모터 값을 업데이트하신다면 사실 DShot600(약 27us 전송)은 오버스펙에 가깝습니다.
 즉, 통신 속도는 매우 빠른데 업데이트는 그보다 훨씬 천천히 일어나는 구조입니다. 
 이 경우 신호 품질이 조금만 나빠도 1ms라는 소중한 제어 주기 내에서 유효한 데이터 전송이 실패할 확률이 높습니다.

RMT 타이밍 미세 조정: 만약 드라이브 강도를 높여도 비프음이 지연된다면,
 RMT 비트 설정에서 High 구간(duration_high)을 1~2틱 정도 늘려서 저항에 의한 전압 상승 지연을 보상해 보시기 바랍니다.

DShot300의 가치: 1ms 루프타임 환경이라면 DShot300(약 54us 전송)으로도 충분히 실시간 제어가 가능합니다.
 만약 앞의 조치들로도 비프음 동기화가 안 된다면,
  비행 안정성을 위해 신호 신뢰도가 높은 DShot300을 적극 고려해 보세요.

지금 이렇게 커지니까 오히려, 회전이 강해진것같습니다. 아니, 좀 뭔가 정제된것같으면서도, 혼란스럽습니다. 잠깐 다시 상황을 정리해보죠. 1. 기체 특성(기본 하드웨어 상태) -> pid를 적용안하고, bias 및 trim을 기본으로 주면, 5시 방향으로 약간의 ccw로 방향으로 회전하면서 상승 2. trim/bias 적용 후 // mixer bias #define FRONT_THRUST_BIAS 0.980f//0.985f #define REAR_THRUST_BIAS 1.020f//1.015f #define LEFT_THRUST_BIAS 1.000f//0.985f 1.007 #define RIGHT_THRUST_BIAS 1.000f//1.015f 0.993 // designated trim float roll_manual_trim_rad_ = DEG2RAD(-0.20f); //trim-- -> roll(-) float pitch_manual_trim_rad_ = DEG2RAD(1.50f); //trim-- -> pitch(-) ->믹서는 앞뒤로 4퍼센트(물론 trim을 제외한, bias로만 잡으려면 더 크게 잡아야함, 8~ 10프로 차이 예상), trim은 최대 1.5도로 줘야 pitch가 빠르게 뒤로(6시방향)가면서 상승하지 않음, 물론 지금 값도 뒤로 상승함. roll은 yaw회전이 너무 심해서 판단 불가,(5시방향 -> roll을 어떤 방향을 줘야할지 판단 안됨) 3. pid 적용 후 -> pid를 적용하면, 어느정도 억제 하는것은 분명함. 그러나 값을 크게해도 잡지 못함. 사실 어떻게든bias/trim을 세부적(0.00x)단위로 잡으면 잡힐 수 는 있다고 생각함, 그러나, 배터리 위치가 조금이라도 변경하면 바로 다시 맞춰야함. 무엇보다, ccw를 전혀 잡지 못하고 있음. 4. (어제) pid authority, output(mixer) authority 재설정 -> throttle 위치를 조정했으나, 근본적인 ccw 회전( 지면 및 공중 모두) 잡지 못하고 있음. 5. yaw feed-forward 적용 -> 별차이 못느낌, 아마 부호가 잘못되었을 가능성 큼 float yaw_ff = 0.0f; float yaw_thrust_ff = 0.0015f; if (throttle_rate_cmd > 0.0f) { yaw_ff = throttle_rate_cmd * yaw_thrust_ff; } yaw_ff = std::clamp(yaw_ff, -0.010f, 0.0f); mix_in.yaw += yaw_ff; -> cw로 가려면 mix_in.yaw에 - 값이 추가해야하나, 현재 -0.010~0.0f가 한계임. -> 하지만 무엇보다, 소프트웨어에서 벗어나는 값(일반적인 센서기반 계산이 아닌, 아웃풋 보정용 계산이라, 후에 어떤 일이 벌어질지 모름 위 내용 정리 해봤습니다. 결국 지금 최우선은 ccw회전 방지입니다. 사실 말하면, 같은 부품에, 비슷한 무게에, 거의 가운데의 무게 중심(cog)인데 ccw회전 이유를 도저히 모르겠습니다.

HandleCommandEvents()
- CMD_ARM_REQUEST 수신 → armed_ = true
- CMD_DISARM_REQUEST 수신 → armed_ = false
- CMD_SOFT_LANDING 수신 → soft_landing_requested_ = true
- CMD_EMERGENCY_STOP 수신 → emergency_stop_requested_ = true 또는 즉시 정지

Disarmed()
- armed_ == true
- ekf_ready_
- imu_bias_ready_
- disarmed_settle_dt_ 조건
→ ARMING

Arming()
- armed_ == true 유지
- throttle low
- arming_check_dt_ > 2s
→ ARMED
- armed_ == false 또는 timeout
→ DISARMED

Armed()
- armed_ == false
→ LANDING(SOFT) 또는 정책에 따라 LANDING
- soft_landing_requested_
→ LANDING(SOFT)
- timeout
→ LANDING(FAILSAFE)
- emergency_stop_requested_
→ DISARMED / motor stop

Landing()
- landing_throttle_ 감소
- 완료 시 DISARMED
